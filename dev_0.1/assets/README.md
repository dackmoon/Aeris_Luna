# Assets

这里用于存放 Luna v0.1 的视觉资产。`Aeris` 是长期项目/体系名称，`Luna` 是当前角色名称。

v0.1 先以图片为主。视频可以保留为参考素材，但不作为第一版必须产出。

## 目录说明

- `raw/`：AI 生图工具直接导出的原始候选图。
- `selected/`：从原始候选图中挑出的方向图。
- `avatar/`：用于聊天界面和小尺寸显示的头像。
- `expressions/`：用于网页 Demo 表情切换的表情图。
- `video-references/`：视频参考素材，例如动作参考、动效参考、未来 Live2D/动画参考。

## 图片规格

v0.1 优先保证安卓手机网页和未来树莓派小屏幕上看得清楚。第一版不建议生成复杂全身图，优先使用头像、胸像和半身像。

### 主形象半身图

- 建议比例：`3:4` 或 `4:5`。
- 推荐分辨率：`1536x2048` 或 `1600x2000`。
- 用途：网页中间展示、角色主视觉、后续表情和动作扩展参考。
- 建议文件名：`main_01.png`、`main_02.png`。

### 头像

- 建议比例：`1:1`。
- 推荐分辨率：`1024x1024`。
- 用途：聊天头像、状态头像、App 图标候选、小屏幕显示。
- 建议文件名：`avatar_01.png`。

### 表情图

- 建议比例：`1:1` 或 `3:4`。
- 推荐分辨率：`1024x1024` 或 `1536x2048`。
- 用途：网页 Demo 表情切换。
- 关键要求：脸部和表情在手机小屏幕上必须清楚。

### 背景或封面图

- 建议比例：`9:16`。
- 推荐分辨率：`1080x1920`。
- 用途：安卓手机竖屏页面背景，可选。
- v0.1 不是必需项。

## 文件格式

- 原始生成图：优先 `PNG`。
- 网页展示图：可以转成 `WebP`。
- 透明背景角色图：优先 `PNG`。
- 参考图：可以使用 `PNG`、`JPG` 或 `WebP`。
- 最终角色素材不建议优先使用 `JPG`，除非只是临时参考。

## 建议命名

主形象：

- `main_01.png`
- `main_02.png`

头像：

- `avatar_01.png`

表情：

- `default.png`
- `happy.png`
- `listening.png`
- `thinking.png`
- `comforting.png`

当前 v0.1 先使用这四个表情文件：

- `expressions/smile.png`：默认微笑。
- `expressions/happy.png`：开心鼓励。
- `expressions/listening.jpeg`：温柔倾听。
- `expressions/thinking.jpeg`：思考好奇。

表情映射记录在 `expressions/manifest.json`。缺少的 `comforting` 安抚关心表情，v0.1 暂时复用 `listening`。

视频参考：

- `wave_reference.mp4`
- `idle_motion_reference.mp4`

## v0.1 文件优先级

第一版优先放图片：

1. 主形象半身图。
2. 头像。
3. 5 个核心表情。

视频不是 v0.1 必需项。等静态形象稳定后，再考虑动作视频、Live2D、口型同步或树莓派屏幕动效。
