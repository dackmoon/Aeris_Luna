# Luna Live2D 实现路线 v0.1

本文档记录 Luna 第一版动态形象的主路线：使用现成已绑定 Live2D 模型，先跑通网页端显示、表情切换、TTS 播放和音频驱动口型。当前 Luna 图片继续作为角色视觉方向参考，暂时不直接拆成 Live2D 模型。`Aeris` 保留为长期项目/体系名称。

## 结论

这个方案可行，且比图生视频更适合作为智能陪伴助手的长期技术路线。

核心原因：

- Live2D 可以实时交互，不是固定视频片段。
- 口型可以由 TTS 音频实时驱动。
- 网页端可以运行，后续能迁移到安卓手机浏览器和树莓派 Chromium 全屏。
- 先使用现成模型可以大幅降低早期成本，避免一开始就做拆图和绑定。

## v0.1 目标

v0.1 不追求 Luna 专属 Live2D 模型，而是先验证技术链路：

1. 在网页中加载一个现成 Live2D 模型。
2. 模型能待机、眨眼、轻微呼吸。
3. 能切换基础表情，例如默认、开心、倾听、思考。
4. 播放一段 TTS 或本地音频。
5. 使用 Web Audio API 分析音频音量。
6. 将音量映射到 Live2D 的 `ParamMouthOpenY`。
7. 实现“声音越大，嘴巴张得越大”的自动口型效果。

## 不做什么

v0.1 暂不做：

- 从当前 Luna 图片手动拆图。
- 从零绘制 PSD 分层立绘。
- 手动 K 帧。
- 精准逐字口型或音素级口型。
- 完整语音识别。
- 长期记忆系统。
- 树莓派硬件部署。

这些可以在技术链路跑通后再做。

## 推荐实现路线

### 第一步：准备现成 Live2D 模型

可以使用：

- 淘宝购买的已绑定 Live2D 模型。
- 免费开源 Live2D 示例模型。
- Live2D Cubism SDK 自带或社区示例模型。

模型要求：

- 必须是已绑定模型。
- 最好包含 `model3.json`。
- 最好支持基础表情文件。
- 必须有嘴巴参数，常见为 `ParamMouthOpenY`。
- 最好有自动眨眼或眼睛相关参数。

当前 Luna 图片用于判断“未来正式模型应该长什么样”，不建议直接拿来拆。

### 第二步：网页加载模型

前端先做一个最小页面：

- 一个全屏或竖屏容器。
- 中间显示 Live2D 模型。
- 底部显示播放按钮或简单对话按钮。
- 可以手动切换表情。

技术选型可以是：

- React 或 Next.js。
- Live2D Cubism Web SDK。
- WebGL / Canvas 渲染。
- Web Audio API 做音频分析。

### 第三步：播放 TTS 或本地音频

v0.1 可以先不用真正接 TTS 服务，先放一段本地音频测试：

- `demo-audio/hello.mp3`
- `demo-audio/encourage.mp3`
- `demo-audio/comfort.mp3`

等口型链路验证成功，再替换成真实 TTS。

### 第四步：Web Audio API 分析音量

基本思路：

1. 使用 `AudioContext` 播放音频。
2. 创建 `AnalyserNode`。
3. 每一帧读取音频数据。
4. 计算 RMS 或平均音量。
5. 将音量归一化为 `0 - 1`。
6. 写入 Live2D 参数 `ParamMouthOpenY`。

概念示例：

```text
audio volume 0.00 -> ParamMouthOpenY 0.0
audio volume 0.30 -> ParamMouthOpenY 0.3
audio volume 0.80 -> ParamMouthOpenY 0.8
```

为了更自然，可以加平滑：

```text
mouthOpen = previous * 0.7 + currentVolume * 0.3
```

### 第五步：表情和状态联动

模型加载成功后，可以把对话状态映射到 Live2D 表情：

- `idle`：默认待机。
- `happy`：开心鼓励。
- `listening`：温柔倾听。
- `thinking`：思考好奇。
- `speaking`：播放语音时嘴巴跟随音量开合。

如果购买的模型没有这些表情，也可以先用模型自带表情验证。

## 技术风险

需要提前注意：

- 现成模型的授权是否允许商用、二次开发、网页展示。
- 不同模型的参数名可能不一致，不一定都叫 `ParamMouthOpenY`。
- 移动端浏览器 WebGL 性能需要测试。
- 安卓浏览器可能限制音频自动播放，需要用户先点击一次。
- 树莓派性能有限，模型复杂度不能太高。
- 音量驱动口型不是精准口型，只是“跟着声音开合”。

## v0.1 验收标准

当满足以下条件时，可以认为 Live2D 技术路线初步可行：

- 安卓 Chrome 可以打开网页并显示模型。
- 模型帧率基本稳定。
- 点击按钮可以播放音频。
- 播放音频时嘴巴会随音量开合。
- 音频结束后嘴巴能自然闭合。
- 可以切换至少 2 个表情或状态。
- 效果比图生视频更适合实时陪伴场景。

## 后续升级

技术链路跑通后，再考虑：

1. 将现有 Luna 视觉方向重制为正式 Live2D 分层立绘。
2. 找画师输出 PSD 分层文件。
3. 找绑定师或使用 Live2D Cubism 绑定。
4. 接入真实 TTS。
5. 接入语音识别。
6. 接入人设、记忆和对话系统。
7. 部署到树莓派全屏网页。
