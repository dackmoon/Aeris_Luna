# Aeris / Luna 实现计划

本文档记录 Luna 从形象设计到可运行自用 App 的长期阶段计划。版本号约定见 [versioning.md](./versioning.md)，已完成版本的详细变更见 [CHANGELOG.md](../CHANGELOG.md)。

当前主路线：混合 App。外壳负责手机权限和设备能力，内部用 WebView 承载月光陪伴体、状态机、对话界面、TTS 播放和声音可视化逻辑。

## 总体路线

```text
形象方向确认
-> Android 混合 App 原型
-> 月光陪伴体实时渲染
-> 状态机调度
-> TTS 播放和声音可视化
-> 麦克风录音和语音识别
-> 对话模型、人设和记忆
-> iOS / 树莓派扩展
```

## 技术选型建议

优先方案：

- App 框架：Capacitor / Ionic + React。
- UI 内核：WebView。
- 角色渲染：月光陪伴体，使用 Canvas 2D 粒子系统实时渲染流动能量体。
- 状态调度：前端状态机，根据用户操作和回复阶段切换光效、粒子、波纹和节奏。
- 说话表现：TTS 播放时用波纹、光晕和音量可视化表达，不追求口型。
- Android 自用：打包 APK 后直接安装。
- iOS 自用：后续用 Xcode 真机安装或 TestFlight。

这个方案的好处是：不需要会画画、拆 PSD、绑定 Live2D 或维护大量视频素材；一致性和流畅性更可控。录音、权限、本地文件等能力由 App 原生层处理，视觉和 UI 沿用网页技术，后续也更容易迁移到树莓派 Chromium 全屏网页。人形视频 loop 和 Live2D 均保留为归档/长期可选路线，不作为当前主路线。

## 阶段 2：v0.3 月光陪伴体原型

目标：用实时渲染的月光体替换人形视频背景，解决黑屏、断裂和一致性问题。

要做的事情：

- 建立 `ParticleSoul` 组件。
- 用高密度粒子、动态向量场、拖尾、发光叠加和呼吸动效表达 Luna。
- 保留原有 6 个状态：`idle`、`listening`、`thinking`、`speaking`、`happy`、`comforting`。
- 状态切换时改变颜色、节奏、粒子运动和波纹样式。
- 保留顶部对话入口、状态下拉框、底部麦克风按钮。
- 暂时不接真实音频，只做模拟录音和状态切换。

视觉要求：

- 一致性强，不出现角色漂移。
- 状态切换无黑屏。
- 动效自然、柔和、适合宝宝陪伴场景。
- 颜色低刺激，避免强烈闪烁。

验收标准：

- Android 手机上月光体显示稳定。
- 6 个状态都能切换。
- 状态切换无黑屏、无播放按钮。
- 动效在手机上保持流畅。

## 阶段 3：v0.4 声音可视化

目标：让 Luna 在录音和说话时有真实的声音反馈。

要做的事情：

- 建立 `audioLevel` 视觉参数。
- 录音时显示更真实的波形。
- TTS 播放时用音量驱动光晕、波纹或粒子强度。
- 为 `speaking` 状态添加扩散波纹。
- 为 `listening` 状态添加向内聚拢的声波反馈。

核心逻辑：

```text
录音音量 -> 波形高度
TTS 音量 -> 光晕强度 / 波纹扩散
emotion -> 状态颜色 / 粒子节奏
```

验收标准：

- 录音时视觉有实时反馈。
- 说话时不需要口型也能感到 Luna 在回应。
- 视觉反馈不刺眼、不打扰。

## 阶段 4：v0.5 TTS 播放与说话状态

目标：播放 Luna 的语音回复时，让月光体进入自然的 speaking 状态。

要做的事情：

- 准备本地测试音频，例如 `hello.mp3`。
- 在 App 中播放音频。
- 播放开始时切换到 `speaking` 状态。
- 播放结束后回到 `idle` 或下一个状态。
- 用 Web Audio API 获取音量，用音量控制亮度、波纹和粒子。

核心逻辑：

```text
开始播放 TTS -> speaking
TTS 播放中 -> 音量驱动波纹和光晕
TTS 结束 -> idle / listening
```

验收标准：

- 点击按钮后能播放语音。
- 播放时月光体进入说话状态。
- 语音结束后能自然回到待机或倾听状态。
- 不需要口型也能让用户感到 Luna 正在回应。

## 阶段 5：v0.6 麦克风录音与语音识别

目标：让用户可以用语音和 Luna 互动。

要做的事情：

- 申请 Android 麦克风权限。
- 实现按住说话或点击录音。
- 保存或上传录音。
- 接入语音识别服务。
- 将识别文本显示在聊天框。
- 根据识别内容触发 Luna 回复。

可选路线：

- 先使用云端语音识别。
- 后续再研究本地语音识别。

验收标准：

- 手机可以录音。
- 录音可以转成文字。
- 识别结果能进入对话流程。
- 失败时有明确提示。

## 阶段 6：v0.7 对话模型、人设和记忆

目标：让 Luna 开始具备基本陪伴对话能力。

要做的事情：

- 把 `character.md` 转成系统提示词。
- 建立基础人设配置文件。
- 接入一个对话模型 API。
- 根据用户输入生成回复。
- 回复时播放 TTS。
- 回复阶段映射到月光体状态，例如 `thinking`、`speaking`、`happy`、`comforting`。
- 建立简单记忆，例如宝宝昵称、兴趣、近期话题。

早期记忆可以使用：

- 本地 JSON。
- SQLite。
- 轻量 key-value 存储。

验收标准：

- Luna 能按人设回复。
- 回复语气温柔、简短、有陪伴感。
- 可以记住少量基础信息。
- 月光体状态、语音和声音可视化能配合对话流程。

## 阶段 7：v0.8 iOS 自用版本

目标：在 iPhone 上验证同一套 App。

要做的事情：

- 配置 iOS Capacitor 项目。
- 使用 Xcode 真机运行。
- 配置麦克风权限。
- 测试 WebView 中月光体实时渲染性能。
- 测试音频播放和状态切换。

注意事项：

- iOS 自用安装比 Android 麻烦。
- 免费 Apple ID 可能有签名时间限制。
- 长期自用可能需要 Apple Developer 账号或 TestFlight。

验收标准：

- iPhone 可以运行 Luna App。
- 麦克风和音频播放正常。
- 月光体动效性能可接受。

## 阶段 8：v0.9 树莓派屏幕版本

目标：把 Luna 迁移到树莓派和自有屏幕上。

要做的事情：

- 准备树莓派系统和屏幕。
- 使用 Chromium 全屏打开 Web 版本。
- 测试月光体实时渲染性能。
- 连接麦克风和扬声器。
- 测试语音输入、TTS 输出和月光体状态切换。
- 配置开机自启动。

验收标准：

- 树莓派开机后自动显示 Luna。
- 可以播放语音并驱动声音可视化。
- 基础交互稳定。
- 长时间运行不会明显卡顿或崩溃。
